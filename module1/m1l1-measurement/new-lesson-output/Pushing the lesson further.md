> **CidiLabs Callout Box (Style: Danger/Alert)**
> **"Lost in Space"**
> 
> In 1999, the Mars Climate Orbiter approached the Red Planet after a 10-month journey. Instead of entering a stable orbit, it vanished into the atmosphere and was destroyed. The disaster wasn't caused by a mechanical failure, but by a unit mismatch: one engineering team used English units (pounds-force), while another used Metric units (Newtons).
> 
> **The Lesson:** In University Physics, a unit error isn't just a "wrong answer"—it is a catastrophic engineering failure. This lesson ensures you have the tools to prevent that.

## [Section 1] The Gold Standards: How We Define Reality
Physics begins with the ability to define the world around us using universal constants that do not change over time.

**🎥 Watch:** Standards of the Universe
- **The Second:** Learn how we use the vibrations of a Cesium-133 atom to define time—a standard essential for the GPS in your phone.
- **The Kilogram:** Explore the shift from "Le Grand K" (the platinum-iridium cylinder) to definitions based on the Planck Constant.
- **Atomic Mass & Density:** Understanding the amu (defined by Carbon-12) and how we measure mass at the atomic scale.

To further enhance your PHYS 2325 course page using CidiLabs Design Plus, you can move beyond standard callout boxes and accordions. For a calculus-based physics course, these elements help bridge the gap between abstract mathematical concepts and physical reality.

Here are several specific CidiLabs elements and activities that would fit perfectly for Module 1 Lesson 1: Measurements:

1. **The "Prefix Mastery" Flip Cards**
	- Use the Flip Card element to help students memorize SI prefixes.
	- *Front of Card:* The Prefix (e.g., "Nano-", "Giga-", "Femto-")

	- Use Modals (pop-up windows) to keep the page clean while providing deep-dive help.
	- *Trigger:* A button labeled "Is that zero significant?"
	- *Content:* A quick flowchart or "Decision Tree" explaining the rules of significant figures.

3. **Dimensional Analysis "Step-by-Step" Progress Bar**
	- Use the Progress Bar or Numbered List style to break down the "Chain-Link" process:
		1. Identify Target Units
		2. List Known Factors
		3. The Algebra Check
		4. The Calculator Check

4. **"Deep Dive" Sidebars for Calculus Context**
	- Use Sidebars (floated to the right) to introduce concepts that will appear in the next lesson.
	- *Title:* "Why Precision Matters for Calculus"
	- *Content:* Explain that as t (time) approaches zero in the next lesson (Instantaneous Velocity), our measurement precision becomes the limiting factor of our derivative.

	- Use a specialized "Instructor Note" Callout Box with a small circular avatar.
	- *Message Example:* "When I was an undergrad, I always forgot that volume conversions like cm³ to m³ require you to cube the conversion factor (100³). Don't let the Herring problem in Section 5 trip you up!"

	- *Button Example:* [Icon: Eye] "Explore the Scale of the Universe" (Linking to the famous Scale of the Universe 2 interactive).

8. **Formula "Cheat Sheet" Quick Link**
	- Provide a downloadable PDF or a pop-up window of the "Standard Conversion Factors" students will need for the entire module.

In the Design Brief, the mention of what CidiLabs tool to use seems almost to be made up names. Since Gemini was used for this, I asked it to clarify what tool may have been meant.

It gave me this response:

# CidiLabs DesignPLUS Tool Verification & Translation

The following analysis compares the generic terms used in the design brief against the actual CidiLabs DesignPLUS tool names and locations.

## 1. "Objectives List"
* **Verdict:** Not a specific tool name.
* **Real Tool:** **Content Blocks**
* **How to find it:** `Add New Elements > Content Blocks`. Select "Objectives" from the list.
* **Note:** The tool defaults to a single line. To create a list, press `Enter` after the first objective; the styling will automatically continue for subsequent items.

## 2. "CidiLabs Button Bar"
* **Verdict:** Not a specific tool name.
* **Real Tool:** **Navigation** (specifically **Module Nav** or **Quick Links**)
* **How to find it:** `Add New Elements > Navigation`.
* **Note:** "Module Nav" creates a bar of buttons (Previous / Home / Next). "Quick Links" creates a bar of custom links that look like buttons.

## 3. "Note Box" & "Success Box"
* **Verdict:** These are **Styles**, not separate tools.
* **Real Tool:** **Callout**
* **How to find it:** `Add New Elements > Callout`.
* **How to configure:**
    * Insert a generic Callout.
    * Use the **Style Editor** (paintbrush icon).
    * For "Note": Select the **Info** style (usually blue).
    * For "Success": Select the **Success** style (usually green).

## 4. "Interactive Block"
* **Verdict:** Too generic.
* **Real Tool:** **Accordions** or **Tabs**
* **How to find it:** `Add New Elements > Accordions / Tabs`.
* **Note:** These are the primary tools used to "hide" content until a user clicks, making the page feel interactive.

## 5. "CidiLabs Checklist Component"
* **Verdict:** Partially correct, but often confused with the editor-only checklist.
* **Real Tool (Student-Facing):** **User Progress**
* **How to find it:** `Add New Elements > Advanced > User Progress`.
* **Note:** The standard "Checklist" tool is often for editors only. The **User Progress** tool inserts items that students can actually check off (saved in their browser cache).

## 6. "Action Buttons"
* **Verdict:** Generic web term.
* **Real Tool:** **Hyperlinks + Button Styles**
* **How to find it:**
    1.  Create a normal link in the Canvas editor.
    2.  Select the link.
    3.  Go to `Current Element Style > Buttons` (or `Style Editor > Buttons`) in the CidiLabs sidebar to apply the visual button style.


### 7. "Step by Step Block"
* **Verdict:** Generic description, not a tool name.
* **Real Tool:** **Module List** (for a list of module items) or **Content Blocks** (specifically the **"Steps"** block).
* **Likely Intention:**
    * If the design calls for a list of module items (e.g., "Step 1: Read this," "Step 2: Submit this"), the tool is **Module List** (`Add New Elements > Module List`).
    * If the design calls for a text-based visual list of steps (non-linked), use **Content Blocks** and select the **"Steps"** block type (`Add New Elements > Content Blocks > Steps`).
