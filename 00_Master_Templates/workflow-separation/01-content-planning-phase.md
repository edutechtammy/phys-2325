# Content Planning Phase - AI Workflow

## 🎯 Purpose
This phase focuses EXCLUSIVELY on content development and pedagogical design. NO technical implementation is involved in this phase.

---

## 📋 Phase 1: Content Planning Only

### ✅ **Content Analysis Tasks**
- [ ] **Audit Review:** Analyze content-audit-findings.html for module-specific issues
- [ ] **Learning Objective Analysis:** Review gaps between stated objectives and needed content
- [ ] **SME Consultation:** Extract insights from video transcripts and consultation guides
- [ ] **Pedagogical Assessment:** Identify learning sequence, difficulty progression, concept connections

### ✅ **Content Development Tasks** 
- [ ] **Learning Sequence Design:** Plan optimal concept progression and connections
- [ ] **Difficulty Scaffolding:** Design practice progression (basic → intermediate → advanced)
- [ ] **Assessment Planning:** Design knowledge checks, practice problems, and application exercises
- [ ] **Real-World Applications:** Identify authentic chemistry applications and case studies
- [ ] **Interactive Element Planning:** Specify what interactions are needed (without technical implementation)

### ✅ **Content Planning Outputs**
Create structured content plans using these templates:

#### **Content Plan Template**
```markdown
# M[X]L[Y] Content Plan: [Lesson Title]

## Learning Objectives
[List specific LOs this lesson addresses]

## Content Sequence
1. **Introduction/Hook** (2-3 minutes)
   - Real-world connection or problem
   - Learning objective preview

2. **Core Concepts** (15-20 minutes)
   - Concept 1: [Description] 
   - Concept 2: [Description]
   - Key connections and relationships

3. **Practice Progression** (15-20 minutes)
   - Basic Level: [Description of practice type]
   - Intermediate Level: [Description of practice type] 
   - Advanced Level: [Description of practice type]

4. **Real-World Applications** (10-15 minutes)
   - Application 1: [Industry/research context]
   - Application 2: [Daily life context]
   - Application 3: [Environmental/health context]

5. **Assessment & Reflection** (5-10 minutes)
   - Knowledge check format
   - Self-assessment approach

## Interactive Elements Needed
- [ ] [Specific interaction type]: [Educational purpose]
- [ ] [Specific interaction type]: [Educational purpose]

## Assessment Strategy
- **Formative:** [Approach]
- **Summative:** [Approach]
- **Self-Assessment:** [Approach]

## Connection Points
- **Previous Lessons:** [How this builds on prior knowledge]
- **Future Lessons:** [How this prepares for upcoming content]
- **Lab Course Coordination:** [Related lab activities/concepts]

## Success Criteria
Students will be able to:
- [Specific measurable outcome 1]
- [Specific measurable outcome 2] 
- [Specific measurable outcome 3]
```

### ✅ **Phase 1 Validation Checkpoint**
Before moving to Phase 2, validate:
- [ ] Content plan addresses all identified audit gaps
- [ ] Learning sequence follows logical progression
- [ ] Practice activities match appropriate difficulty levels
- [ ] Real-world applications are authentic and relevant
- [ ] Assessment approach aligns with learning objectives
- [ ] **NO technical implementation details are included**

---

## 🚫 **Phase 1 Restrictions**
**DO NOT include in Phase 1:**
- HTML code or Canvas formatting
- Specific template selections
- CidiLabs component specifications
- File naming or technical structure decisions
- Canvas constraint considerations

**Focus ONLY on:**
- What content is needed
- How concepts should be sequenced
- What pedagogical approaches are most effective
- How students will practice and apply knowledge
- What assessments are appropriate

---

## ➡️ **Transition to Phase 2**
Once content planning is complete and validated:
1. Save content plan as `m[X]l[Y]-content-plan.md`
2. Mark Phase 1 as complete
3. Hand off to Phase 2 for technical implementation
4. **No content changes** should occur during Phase 2
